VetTrack native mobile redesign preview

Phase 1 design artifact for reviewing the future Expo UI before implementation. RTL/Hebrew is treated as primary; visible copy is represented as i18n key labels so implementation can stay `t()`-driven.

Design System Direction

Mobile tokens evolve the existing VetTrack web brand and remove Expo-starter blue. Status colors are first-class, including critical and needs_attention.

Brand + surfaces

Primary actions, active navigation, selected filters, content background, and sheets/cards.

Status taxonomy

ok, issue, maintenance, sterilized, critical, needs_attention. No collapsing critical states to info.

Type + density

Display 28/34, title 22/28, headline 17/22, body 15/22, subhead 13/18, caption 12/16. DM Mono for counts, timers, IDs.

Motion + safety

120-220ms state feedback, sheet present, scan transitions, sync queued-to-synced, return undo. Reduced motion keeps all state visible.

Core Components

Specimens for implementation vocabulary: 44-48px targets, restrained elevation, clear state variants, RTL-safe leading/trailing structure.

Route Coverage

Every route from the brief is accounted for. Some are combined in one phone frame when the implementation should share a native surface, such as a sheet over a detail screen.

Auth + root

`app/_layout.tsx`: splash/font/RTL readiness · `app/+html.tsx`: no native UI · `app/+not-found.tsx`: not-found frame · `(auth)/_layout.tsx` + `(auth)/sign-in.tsx`: auth frame and onboarding sheet.

App frame

`(app)/_layout.tsx`: auth guard and banner priority · `(tabs)/_layout.tsx`: tab bar, scan FAB, stack headers, safe-area behavior.

Tabs

`index.tsx`: Today · `equipment.tsx`: equipment list · `my-equipment.tsx`: custody list · `rooms.tsx`: rooms list · `alerts.tsx`: grouped alerts · `auth.tsx`: Account/Settings.

Stacks + sheets

`scan.tsx`: NFC state machine · `equipment/[id].tsx`: detail · `equipment/[id]/update-status.tsx`: status sheet · `equipment/new.tsx`: grouped form · `rooms/[id].tsx`: room detail · `shift/handoff.tsx`: handoff.

Cross-cutting states

Emergency entry/status/blocked states, offline queue states, slow network skeletons, stale data banners, conflict/sync-failed, and deep-link/NFC resolving.

Implementation notes: use Expo Router native stacks and tabs, `useSafeAreaInsets`, existing `expo-symbols` icon surface, no new npm package required for this design direction. The dark sample maps to the existing forest-dark theme, not a separate visual language.
